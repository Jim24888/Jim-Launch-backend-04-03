📌 转化归因管理机制

此处展示系统内已有的所有转化归因资产。

  • 由于业务需求，列表中隐藏了“媒体账号”及“深度强化目标ROI”列。
  • 点击上方的 【+ 新建转化归因模版】，可通过弹窗快速复用资产配置。

配置好的模板将沉淀在库中，供“上计划”时批量新建并应用。

📌 核心机制说明（必读）

为降低配置出错率，模版创建全面采用「强制复用」机制。您只需选择已跑通的媒体账号及其成熟转化，系统将自动拉取并锁定所有底层参数，您只需命名模版即可保存。

  • 全局新建防重机制：在新建广告计划并应用此模板时，系统会统一提取这套已锁定的模版参数，为当前账号统一发起“新建转化归因”操作。依赖媒体端API幂等性自动防重，即使重复提交也只会返回同一个资产ID，安全高效。

📝 逐字段配置说明

一、 账号与模版识别

  • 选择媒体账号：下拉拉取系统内授权的媒体账号。
  • 选择转化归因：选中账号后拉取该账号下的可用转化。选中后，系统将映射下方所有信息并设为只读不可更改状态。
  • 转化归因模版名称：唯一允许用户填写的字段，用于在平台上计划时搜索识别本模版。

二、 自动带入只读字段

以下字段均由系统拉取源转化自动填充并锁定，仅供核对：

  • 转化场景与载体：Android、iOS、鸿蒙或微信小游戏，以及对应的应用/小游戏ID。
  • 数据上报与归因设置：上报方式(API/SDK)、归因方式(点击/激活/注册)、数据源ID、点击监测链接。
  • 优化目标设定：包含浅度目标及深度的行为或ROI目标。

🚀 上计划 - 模版应用机制

【转化归因模版 选择与搜索】

  • 下方的下拉框支持直接输入文字进行模糊搜索，体验更顺畅。

【全局新建与自动防重机制】

  • 待上计划时选择了该模版，系统底层会自动提取模版参数，为当前所有选中的媒体账号统一发起“新建转化归因”操作。
  • 借助媒体端 API 的幂等特性（相同参数创建返回同一资产 ID），系统自动实现无缝防重，绝不会产生重复资产，同时也彻底免去了前置查询的低效动作。
  • 应用位置：草稿模版、上计划页（单账号与多账号）。
